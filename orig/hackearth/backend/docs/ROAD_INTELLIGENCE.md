# Road Intelligence Layer

This document describes the Road Intelligence module added to the AI Data Foundation. It extends physical hotspot locations with deep GIS attributes without relying on random number generation or mock endpoints.

## 1. Architecture

The pipeline consists of modular deterministic engines feeding into a centralized service.

1. **Information Engine (`backend/ai/road_information.py`)**: Designed to query MapMyIndia. Degrades safely to deterministic string heuristics derived from the uploaded CSV dataset `location_name` if keys are missing.
2. **Hierarchy Engine (`backend/ai/road_hierarchy_engine.py`)**: Maps categories into standard smart city hierarchy ranks and assigns priority scoring (e.g., Expressway: 100, Residential: 40).
3. **Width Engine (`backend/ai/road_width_engine.py`)**: Provides deterministic width estimations ranging from 6m (Service) to 30m (Expressway).
4. **Capacity Engine (`backend/ai/road_capacity_engine.py`)**: Calculates accurate lane counts (using standard 3.5m lane width) and Passenger Car Unit (PCU) efficiency limits.
5. **Geometry Engine (`backend/ai/road_geometry_engine.py`)**: Creates reproducible GeoJSON geometries (LineStrings and bounding boxes) representing the hotspot topology.
6. **Confidence Engine (`backend/ai/road_confidence_engine.py`)**: Calculates the algorithmic confidence bounds of all extracted data.

## 2. Pipeline Orchestration (`backend/services/road_service.py`)

The orchestration layer parses `hotspots_with_density.json` and runs every hotspot sequentially through the AI engines. 

It outputs to `backend/data/processed/road_intelligence.json` producing fully featured Pydantic-validated road models. 

## 3. Engineering Assumptions

- **Lane Width**: Modeled at $3.5$ meters per lane (Indian Standard).
- **Capacity**: Base capacity is 1200 PCU / hr / lane. Modifiers apply depending on road hierarchy (expressways flow better per lane than local roads).
- **Line Generation**: Pseudo-line generation maps either East-West or North-South bounding box trajectories based strictly on geospatial hashing parameters to maintain determinism.

## 4. Future Integration
The outputs (`road_priority_score`, `estimated_road_width`, `capacity_score`) generated by this layer are specifically designed to feed into the upcoming **Effective Width Loss Engine** and **Congestion Severity Index (CSI)** calculations.
